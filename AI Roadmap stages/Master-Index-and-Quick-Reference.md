### Chunking Strategy Selection Guide

| Document Type | Best Strategy | Chunk Size | Overlap |
|--------------|---------------|------------|---------|
| Long technical docs | Parent-child | Parent 800, Child 150 | 10% |
| FAQ / Q&A | Semantic | 300-400 | 20% |
| Legal contracts | Contextual | 400-500 | 15% |
| Code files | Recursive (by function) | 500 | 5% |
| News articles | Semantic | 350 | 10% |
| Short product descriptions | Fixed | 200 | 0% |

### Vector Database Selection

| Use Case | Recommended | Why |
|----------|-------------|-----|
| Default/MERN stack | pgvector (Supabase) | SQL joins, no extra service |
| > 10M vectors | Pinecone | Managed, scales well |
| Self-hosted, privacy | Qdrant | Rust performance |
| Prototyping | Chroma | Easy local setup |

### Model Routing Decision Tree

```
Query incoming
    │
    ├── Contains PII/sensitive data? → Local Ollama (never leaves server)
    │
    ├── Simple: yes/no, factual, < 50 words? → Claude Haiku / GPT-4o Mini
    │
    ├── Code generation/review? → GPT-4o
    │
    ├── Complex reasoning, multi-step? → Claude Opus / o1
    │
    └── Default → Claude Sonnet 4
```

### RAGAS Quality Thresholds

| Metric | Target | Below This = Problem |
|--------|--------|---------------------|
| Faithfulness | > 0.85 | Hallucinating — model adds facts not in context |
| Answer Relevancy | > 0.80 | Off-topic — not answering the actual question |
| Context Precision | > 0.75 | Retrieving irrelevant chunks |
| Context Recall | > 0.80 | Missing relevant information in retrieval |

### Redis Pattern → Use Case

| Pattern | Use For |
|---------|---------|
| String + TTL | Simple cache, rate limit counter |
| Hash | Session data, user preferences |
| Sorted Set | Sliding window rate limit, priority queue |
| List | Conversation history buffer |
| Set | Unique membership (indexed documents) |
| Pub/Sub | Real-time progress updates |
| Streams | Durable message queue (at-least-once) |
| SET NX EX | Distributed lock |

### Production Alert Thresholds

| Alert | Threshold | Action |
|-------|-----------|--------|
| Single request cost | > $0.50 | Investigate runaway prompt |
| Daily cost | > 80% of budget | Review usage |
| LLM latency p99 | > 10s | Check provider status |
| Error rate | > 2% | Trigger fallback chain |
| RAG faithfulness | < 0.70 | Review chunking/retrieval |
| Cache hit rate | < 30% | Review cache configuration |
| Queue depth | > 1000 | Scale workers |

## 🛠️ Complete Tech Stack Reference

```
FRONTEND
  Framework:    Next.js 14+ (App Router)
  UI Library:   Tailwind CSS + shadcn/ui
  AI Streaming: Vercel AI SDK (useChat, useCompletion)
  Auth:         Clerk
  Hosting:      Vercel

BACKEND
  Runtime:      Node.js 20+
  Framework:    Express or Fastify
  Language:     TypeScript (strict mode)
  Queue:        BullMQ + Redis (ioredis)
  Logging:      Pino
  Hosting:      Railway or Render

DATABASES
  Primary:      PostgreSQL via Supabase (includes pgvector!)
  Sessions:     Redis (Upstash for serverless)
  Documents:    MongoDB (conversation history)
  Files:        Supabase Storage or AWS S3

AI & ML
  LLM Primary:  Anthropic Claude (claude-sonnet-4-20250514)
  LLM Fallback: OpenAI GPT-4o
  LLM Routing:  LiteLLM proxy
  Embeddings:   OpenAI text-embedding-3-small
  Vector DB:    pgvector (default) → Pinecone (scale)
  Agents:       LangGraph (stateful) or raw API (simple)
  Memory:       Mem0 (cross-session)
  Local Models: Ollama

BILLING
  Payments:     Stripe (subscriptions + usage-based)
  Metering:     Custom token tracking in PostgreSQL
  Invoicing:    Stripe automatic

OBSERVABILITY
  AI Traces:    Langfuse (self-hosted)
  Metrics:      Prometheus + Grafana
  Errors:       Sentry
  Uptime:       Better Uptime
  Analytics:    PostHog

FEATURE FLAGS
  Primary:      Growthbook (self-hosted, open-source)
  Alternative:  LaunchDarkly (managed, expensive)

SECURITY
  PII Detection: Presidio (Microsoft, Python sidecar)
  API Keys:     AWS Secrets Manager or Doppler
  Content Mod:  OpenAI Moderation API

EVALS
  CI Evals:     PromptFoo (YAML-based, CI integration)
  RAG Quality:  RAGAS (faithfulness, relevancy, precision, recall)
  LLM Judge:    Custom implementation (Claude-as-judge)

CI/CD
  Pipeline:     GitHub Actions
  Containers:   Docker (multi-stage builds)
  Deploy:       Vercel (frontend) + Railway (backend) + Modal (GPU)

FINE-TUNING
  Cloud:        OpenAI fine-tuning API
  Open Source:  Unsloth + LoRA on Modal or Runpod
  Serving:      Ollama (local) or vLLM (production GPU)
```

## 🚩 Red Flags — Anti-Patterns to Avoid

| Anti-Pattern | Why It's Bad | What to Do Instead |
|-------------|--------------|-------------------|
| Hardcoded prompts in code | Can't deploy prompt changes without code deploy | Store in DB with versioning |
| `any` type in TypeScript | Silent runtime failures from LLM output | Strict mode + Zod validation |
| No eval suite | Don't know when quality degrades | 50+ case eval in CI |
| Synchronous LLM calls for expensive tasks | Timeouts, poor UX | BullMQ async queue |
| No cost tracking | $10,000 surprise bill | Track tokens per request to DB |
| Single provider with no fallback | Provider outage = total outage | 3-provider fallback chain |
| No tenant isolation testing | Cross-tenant data leakage | Write isolation tests |
| Generic AI developer positioning | Compete with 100,000 others | Pick one niche, go deep |
| 20 half-finished projects | No proof you can ship | 3 polished projects with live demos |
| "I tested it manually" | No regression detection | Automated eval suite |
| Vibes-based prompt changes | No data on what improved | Measure before/after every change |
| No structured logging | Can't debug production issues | Pino with queryable fields |
| Forgetting about security | PII leakage, injection attacks | OWASP LLM Top 10 |
| Fine-tuning first | Expensive, often unnecessary | Exhaust prompting + RAG first |
| Agent without timeouts | Infinite loop, infinite cost | Max iterations + cost limits |

## 💡 The Mindset Shifts That Matter Most

### From Junior to Senior AI Engineer

| Junior Thinks | Senior Thinks |
|--------------|---------------|
| "How do I make the LLM do X?" | "How do I make the system reliably do X and know when it stops?" |
| "It works on my machine" | "It works under load, with failures, and at 2am" |
| "I tested it manually" | "The eval suite tests it automatically on every PR" |
| "Add more to the prompt" | "Budget the context window deliberately" |
| "Use GPT-4o for everything" | "Route to the cheapest model that can do the job" |
| "Ship it" | "What's the rollback plan if quality drops?" |
| "It hallucinated" | "My retrieval is missing the right context — fix chunking" |
| "The AI is broken" | "My faithfulness score is 0.6 — I'm missing groundtruth" |
