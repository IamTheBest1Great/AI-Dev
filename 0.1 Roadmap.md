## 🟢 Phase 1 — AI Foundations
> *Goal: Talk fluently to AI models, control costs, and ship streaming UIs fast*

### Core Skills

**API Basics**
- OpenAI / Anthropic / Google Gemini / Mistral API setup
- API keys, rate limits, token counting, cost management
- Chat completions, streaming responses (SSE), token budgeting
- System prompts vs user prompts — when each matters
- Conversation history (the message array pattern)
- Getting structured JSON output from AI (json_mode, response_format)
- Vercel AI SDK (`useChat`, `useCompletion`, `useObject` hooks)

**Prompt Engineering (Most developers skip the depth here)**
- Zero-shot, one-shot, few-shot prompting
- Chain-of-thought (CoT) and step-by-step reasoning
- Tree-of-thought for complex decisions
- Role prompting and persona engineering
- Negative prompting ("do NOT do X")
- Delimiter techniques (XML tags, triple backticks) for reliable parsing
- Prompt injection awareness and defense
- Prompt compression — say more with fewer tokens

**Cost & Performance**
- Prompt caching (Anthropic / OpenAI) — save 80%+ on repeated system prompts
- Token optimization strategies (trim, compress, summarize old messages)
- Model tiering — GPT-4o Mini vs GPT-4o vs Claude Haiku vs Sonnet
- LiteLLM — one SDK to call any AI model, switch without rewriting code
- Streaming for perceived speed (users hate waiting)

**What Most Developers Miss**
- Dynamic system prompts — inject user data, role, and context at runtime
- Output validation before sending to the user (hallucination defense)
- Prompt versioning from Day 1 (don't hardcode prompts in code)
- A/B testing prompts — same product, different prompts, measure quality
- Model fallback chains — if GPT-4o fails, auto-retry with Claude Sonnet

## 🟡 Phase 2 — Core AI Patterns
> *Goal: Build the backbone of real AI products — search, memory, retrieval, and structured AI output*

### Core Skills

**Embeddings & Vector Search**
- What embeddings are (text → dense numerical vectors)
- Generating embeddings: OpenAI `text-embedding-3-small`, Cohere, local models
- Cosine similarity, dot product — how similarity search actually works
- Chunking strategies: fixed-size, recursive, semantic, by sentence, by token
- Chunk overlap — why it matters and how to tune it
- Metadata filtering on top of vector search
- Namespace/tenant isolation in vector stores

**Vector Databases**
- Pinecone (managed, production-grade)
- Supabase pgvector (PostgreSQL native, great for MERN stacks)
- Weaviate, Qdrant, Chroma (self-hosted options)
- When to use each — cost vs control vs scale tradeoffs

**RAG (Retrieval-Augmented Generation)**
- Basic RAG: upload → chunk → embed → store → retrieve → generate
- Advanced RAG techniques:
  - HyDE (Hypothetical Document Embeddings) — generate a fake answer, embed it, search
  - Re-ranking with Cohere Rerank or cross-encoders
  - Multi-query retrieval — LLM generates 3 query variants, merge results
  - Contextual compression — trim retrieved chunks before sending to LLM
  - Parent-child chunking — store small chunks, retrieve their full parent document
  - Hybrid search: vector (semantic) + BM25 (keyword) combined with RRF scoring
  - Self-querying retrievers — LLM generates metadata filters from natural language

**Structured Outputs & Validation**
- Zod schemas for type-safe LLM output
- Instructor.js / instructor-js for reliable structured extraction
- JSON mode vs tool use for structured data
- Retry on parse failure (with exponential backoff)
- Output validation pipelines before serving to users

**Frameworks**
- LangChain.js — chains, retrievers, document loaders, memory, agents
- LlamaIndex.ts — data ingestion, indexing, query engines
- When to use a framework vs build raw (performance, debugging, vendor lock-in)

**Tool Calling / Function Calling**
- How the tool call cycle works (LLM → tool schema → execute → result → LLM)
- Defining robust tool schemas with descriptions and examples
- Multi-tool calls in a single LLM response
- Parallel vs sequential tool execution
- Error handling when tools fail

**Knowledge Graphs (Advanced)**
- Neo4j for entity-relationship storage
- LLM-extracted entities → knowledge graph
- GraphCypherQAChain for natural language graph queries
- Graph-augmented RAG: vector search + graph traversal combined
- Multi-hop reasoning with knowledge graphs

**What Most Developers Miss**
- Chunking is the #1 RAG failure point — experiment relentlessly
- Evaluating RAG quality (RAGAS framework — faithfulness, relevancy, context precision)
- Document pre-processing: OCR, table extraction, image captioning before embedding
- Incremental indexing — update the vector store without full re-embedding
- Index freshness strategies for live data (event-driven re-embedding)

## 🔴 Phase 4 — Agents & Automation
> *Goal: Build AI that takes actions in the real world, not just responds*

### Core Skills

**Agent Fundamentals**
- Agents vs chatbots — what "agency" actually means
- ReAct pattern: Reason → Act → Observe loop
- Tool-use loops: when to stop vs keep going
- Agent memory: in-context, external (Mem0), knowledge graph
- Agent failure modes: infinite loops, hallucinated tool calls, stuck states
- Human-in-the-loop checkpoints for high-stakes actions

**Frameworks**
- LangChain.js agents — zero-shot ReAct, structured agents
- LangGraph — stateful agent workflows with branching, cycles, checkpoints
- OpenAI Assistants API + Threads + File Search (hosted agent infrastructure)
- CrewAI.js — multi-agent with roles, goals, and delegation

**Browser & Web Automation**
- Puppeteer / Playwright — browser control, scraping, form fill
- Browserbase — cloud-hosted browser sessions for agents at scale
- Stagehand — AI-native browser automation (describe what to do, not how)
- Firecrawl / Jina Reader — clean markdown extraction from any URL
- Exa API / Tavily — AI-optimized web search with full page content

**Data Pipelines**
- Web scraping + AI to clean and structure messy data
- n8n self-hosted workflow automation (visual + code nodes)
- Data extraction pipelines: scrape → extract → validate → store
- Self-healing pipelines: detect extraction failure, retry with different strategy
- Change detection — alert only when something actually changed

**Advanced Agent Patterns**
- MCP (Model Context Protocol) — connect agents to any tool/data source via standard protocol
- Computer Use API (Anthropic) — AI controls a real screen (mouse + keyboard)
- Code generation agents — write, execute, debug code in a loop
- Parallel agent execution for speed
- Agent evaluation — replay conversations, measure task success rate

**What Most Developers Miss**
- Agents need timeouts and circuit breakers — don't let them run forever
- Log every agent step to a trace store (LangSmith / Langfuse) from day one
- Agent security: sandboxed code execution, URL allowlists, action confirmation
- Tool descriptions are prompts — bad descriptions = bad agent decisions
- Test agents with adversarial inputs before giving them real credentials

## 🔵 Phase 5 — Advanced AI Systems
> *Goal: Build multimodal, voice, and multi-agent AI products*

### Core Skills

**Voice AI**
- Whisper API (speech-to-text) + streaming transcription
- OpenAI TTS / ElevenLabs / Cartesia (text-to-speech)
- Real-time voice: OpenAI Realtime API over WebSockets
- Voice Activity Detection (VAD) — know when the user stops speaking
- Speaker diarization — who said what in a meeting
- Noise cancellation preprocessing for better transcription

**Vision & Multimodal**
- Sending images to GPT-4o Vision / Claude / Gemini
- Multi-image reasoning (before/after, comparison, sequences)
- Document vision: extract tables, forms, handwriting from images
- Video understanding (frame extraction + AI analysis)
- Image generation: DALL-E 3, Stable Diffusion via API, Flux
- Image editing: inpainting, outpainting, style transfer

**Multi-Agent Systems**
- Orchestrator + worker agent pattern
- LangGraph stateful workflows (DAGs with cycles)
- CrewAI roles: researcher, writer, editor, publisher
- Agent communication patterns: shared state vs message passing
- Hierarchical agents: manager delegates to specialists
- Agent checkpointing and resumability (long-running tasks)

**Memory Systems**
- Mem0 — persistent cross-session memory layer
- Entity extraction → structured memory (user facts, preferences)
- Memory compression: summarize old conversations into facts
- Memory retrieval: semantic search over the memory store
- Forgetting mechanisms: privacy, memory limits, staleness

**Edge & On-Device AI**
- Transformers.js — run models in the browser (zero latency, full privacy)
- WebLLM — LLMs running directly in Chrome via WebGPU
- ONNX Runtime Web — optimized model inference in browser
- Ollama — local model serving for desktop apps
- React Native on-device AI (phi-3-mini, TinyLlama)

**What Most Developers Miss**
- Voice UI requires interruption handling — users don't wait for AI to finish
- Multi-agent systems need a shared context store (not just passing messages)
- Image understanding quality depends heavily on image resolution and prompt
- On-device models need quantization — 4-bit is usually the sweet spot
- Realtime API billing is per minute, not per token — architect accordingly

## ⚫ Phase 6 — Production & Scale
> *Goal: Ship reliable AI products that survive real users and real load*

### Core Skills

**Observability**
- LangSmith — trace every LLM call, replay conversations, debug
- Langfuse (open-source LangSmith alternative, self-hostable)
- Helicone — cost analytics, latency tracking, request logging
- OpenTelemetry for AI — bring AI traces into your existing APM
- Structured logging: log model, prompt version, tokens used, latency per request

**Evals (The skill 95% of AI devs lack)**
- What evals are: automated quality tests for AI outputs
- LLM-as-judge: use a stronger model to grade your model's output
- RAGAS framework: measure RAG quality (faithfulness, answer relevancy, context recall)
- PromptFoo: write eval suites in YAML, run in CI/CD
- Regression testing prompts: catch when model updates break your product
- Red-teaming your own AI: jailbreak attempts, adversarial inputs, edge cases
- Human eval baselines: build a small labelled dataset, measure against it

**Cost Optimization**
- Prompt compression libraries (LLMLingua) — reduce tokens without losing meaning
- Response caching (Redis) — cache identical or near-identical queries
- Semantic caching (GPTCache) — cache by meaning, not exact match
- Model routing: classify query complexity → send to cheapest capable model
- Batch API for non-real-time tasks (50% cheaper on OpenAI)
- Custom model routing with LiteLLM proxy

**Reliability**
- Retry with exponential backoff on API errors
- Fallback chains: primary model → secondary model → graceful degradation
- Circuit breakers for AI providers (stop hammering a failing API)
- Queue-based AI processing (BullMQ) — never drop a request under load
- Graceful degradation: show cached or default response when AI is down
- Rate limiting per user per day (protect costs, prevent abuse)

**Security for AI Products**
- Prompt injection defense (system prompt hardening, output validation)
- PII detection before sending user data to external AI APIs
- Data residency: know which data leaves your region
- Content moderation (OpenAI Moderation API, Perspective API)
- Secrets management for AI provider keys (never in code)
- OWASP Top 10 for LLM Applications

**What Most Developers Miss**
- Evals before shipping, not after getting bad reviews
- You need a baseline dataset of expected good outputs — start collecting from day 1
- Prompt versioning in a DB, not in code — deploy prompt changes without redeploys
- AI cost alerts before you get a $10,000 surprise bill
- Multi-tenant data isolation: one client's data must never appear in another's RAG

## 🟣 Phase 7 — Cutting Edge & Niche Moats
> *Goal: Stay ahead of 99% of developers with capabilities few have mastered*

### Core Skills

**MCP (Model Context Protocol)**
- MCP architecture: hosts, clients, servers, tools, resources, prompts
- Building your own MCP server (Node.js) — expose any data source or API to Claude
- Connecting MCP to Claude Desktop, Claude Code, custom AI apps
- Publishing MCP servers to the ecosystem
- MCP security model and permission system

**Fine-Tuning & Custom Models**
- When to fine-tune vs prompt engineer (fine-tuning is not always the answer)
- OpenAI fine-tuning: JSONL dataset prep, training, evaluation
- LoRA / QLoRA: parameter-efficient fine-tuning for LLaMA, Mistral, Gemma
- Unsloth: faster fine-tuning on consumer GPUs
- PEFT (Parameter-Efficient Fine-Tuning) patterns
- Dataset curation: quality over quantity, deduplication, formatting
- Evaluating fine-tuned models vs base model (don't trust vibes)
- Serving fine-tuned models: Ollama, vLLM, Hugging Face Inference Endpoints

**Autonomous Agent Loops**
- Human-in-the-loop at critical decision points
- Long-horizon tasks with checkpointing and resumability
- Self-correcting agents: detect failure, re-plan, retry
- Agent memory + tool selection improving over time (few-shot from memory)

**Edge Cases & Niche Skills That Pay Well**
- Document intelligence: AWS Textract / Azure Document Intelligence + AI post-processing
- AI for code: AST-aware code generation, semantic code search, test generation
- Multimodal RAG: embed images alongside text, search across both
- Audio AI: music analysis, voice cloning detection, podcast chapter detection
- Geospatial AI: embedding location context, AI for mapping and routing
- AI for accessibility: live captioning, alt-text generation, readability scoring

**What Most Developers Miss**
- Fine-tuning is expensive — exhaust prompt engineering, RAG, and few-shot first
- Published MCP servers get you visibility in the Claude ecosystem
- Niche domain expertise + AI beats generic AI knowledge every time
- Local model hosting = zero API costs, full privacy = enterprise-grade selling point

## 💼 Phase 8 — The Business Layer
> *The layer that turns technical skills into income — most roadmaps skip this entirely*

### Why This Phase Matters

You can build everything above and still not make money if you can't position, price, and sell your work. This phase is what turns you from "an AI developer" into a **highly paid AI developer**.

### Skills

**Productizing AI**
- Identify the 5 highest-value AI use cases per industry (healthcare, legal, real estate, e-commerce, SaaS)
- AI ROI framing: "saves X hours/week = $Y/month" beats "uses GPT-4"
- MVP scoping: what's the minimum AI that solves the pain?
- Feature vs product distinction: a chatbot is a feature, a client acquisition system is a product

**Freelancing & Consulting**
- Upwork / Toptal positioning for AI engineers
- How to write proposals that win AI projects
- Discovery calls: extract requirements from non-technical clients
- Retainer models: ongoing AI maintenance vs one-time build
- What to charge: AI builds ($3k–$30k), AI SaaS ($500–$5k/month retainer)

**Building AI SaaS**
- Niche down: "AI for dentists" beats "AI chatbot for businesses"
- Pricing models: per-seat, usage-based, credit packs, enterprise contracts
- AI SaaS tech stack: Next.js + Clerk + Stripe + Supabase + OpenAI
- Launch strategies: Product Hunt, Indie Hackers, LinkedIn, Reddit
- Churn prevention: AI products must keep improving or users leave

**Building in Public**
- Document your AI builds on LinkedIn / X / YouTube — weekly
- Open-source a useful AI tool, get GitHub stars, convert to leads
- Write about what you learned (not just what you built)
- Contribute to LangChain.js, Vercel AI SDK, or LlamaIndex — your name in the commits

## 🚩 Red Flags — What Not To Do

- **Don't build tutorial projects, build real ones.** "Clone of X" is forgettable. Solve a real pain.
- **Don't skip evals.** Every serious AI product needs automated quality measurement.
- **Don't hardcode prompts in code.** Store them in a DB and version them.
- **Don't call it "AI-powered" everywhere.** Show the outcome: "saves 3 hours/week" not "uses GPT-4."
- **Don't tune on vibes.** Measure before and after every prompt or model change.
- **Don't build 20 half-finished projects.** Three polished, deployed, documented projects beat twenty repos.
- **Don't ignore security.** Prompt injection and PII leakage are real. Learn OWASP LLM Top 10.
