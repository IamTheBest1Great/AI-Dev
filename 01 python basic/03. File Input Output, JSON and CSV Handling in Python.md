## 1. Basic File I/O in Python

Python provides built-in functions to work with files. The most important is `open()`.

### Opening a File
```python
file_object = open(filename, mode)
```
**Modes**:
- `'r'` – read (default)
- `'w'` – write (creates new file or truncates existing)
- `'a'` – append (adds to end of file)
- `'x'` – exclusive creation (fails if file exists)
- `'b'` – binary mode (e.g., `'rb'` for reading binary)
- `'t'` – text mode (default, e.g., `'rt'`)

**Always close the file** after use: `file_object.close()`. Better yet, use a `with` statement – it automatically closes the file, even if an error occurs.

### Reading Text Files
```python
# Read entire content
with open('example.txt', 'r') as file:
    content = file.read()
    print(content)

# Read line by line
with open('example.txt', 'r') as file:
    for line in file:
        print(line.strip())  # strip removes newline

# Read all lines into a list
with open('example.txt', 'r') as file:
    lines = file.readlines()
    # lines is a list of strings with newlines
```

### Writing Text Files
```python
# Write (overwrites)
with open('output.txt', 'w') as file:
    file.write("Hello, AI!\n")
    file.write("This is a new line.")

# Append
with open('output.txt', 'a') as file:
    file.write("\nAppending more text.")
```

### Example: Save and Load a Simple List
```python
# Save a list of numbers to a text file
data = [1, 2, 3, 4, 5]
with open('data.txt', 'w') as f:
    for num in data:
        f.write(str(num) + '\n')

# Read back
loaded = []
with open('data.txt', 'r') as f:
    for line in f:
        loaded.append(int(line.strip()))
print(loaded)  # [1, 2, 3, 4, 5]
```

**Why for AI**: You'll often save preprocessed data, training logs, or simple configurations in text files.

## 2. Working with JSON

JSON (JavaScript Object Notation) is a lightweight format for exchanging data. It's human-readable and widely used in APIs, configuration files, and datasets.

Python's `json` module provides easy encoding (serialization) and decoding (deserialization).

### Key Functions
- `json.dump(obj, file)` – write Python object to a JSON file
- `json.dumps(obj)` – return JSON string from Python object
- `json.load(file)` – read JSON file and convert to Python object
- `json.loads(json_string)` – parse JSON string and return Python object

### Mapping Between Python and JSON
| Python | JSON |
|--------|------|
| dict | object |
| list, tuple | array |
| str | string |
| int, float | number |
| True | true |
| False | false |
| None | null |

### Examples
```python
import json

# Python dictionary
person = {
    "name": "Alice",
    "age": 30,
    "city": "New York",
    "has_license": True,
    "children": None,
    "scores": [95, 88, 92]
}

# Write to JSON file
with open('person.json', 'w') as f:
    json.dump(person, f, indent=4)  # indent for pretty printing

# Read from JSON file
with open('person.json', 'r') as f:
    loaded_person = json.load(f)
print(loaded_person)

# Convert to JSON string
json_string = json.dumps(person, indent=2)
print(json_string)
```

**Why for AI**: Many datasets are stored as JSON (e.g., COCO annotations, dialogue datasets). You'll also save model hyperparameters, training configurations, and evaluation metrics in JSON format.

## 3. Working with CSV

CSV (Comma-Separated Values) is a simple file format for tabular data. It's the go-to format for datasets in machine learning (e.g., Kaggle competitions).

Python's `csv` module provides functionality to read and write CSV files.

### Reading CSV Files
```python
import csv

# Read as list of rows
with open('data.csv', 'r') as f:
    reader = csv.reader(f)
    header = next(reader)  # get first row (column names)
    for row in reader:
        print(row)  # each row is a list of strings

# Read as list of dictionaries (first row becomes keys)
with open('data.csv', 'r') as f:
    reader = csv.DictReader(f)
    for row in reader:
        print(row['name'], row['age'])  # access by column name
```

### Writing CSV Files
```python
import csv

# Write from list of lists
data = [
    ['name', 'age', 'city'],
    ['Alice', 30, 'New York'],
    ['Bob', 25, 'London'],
    ['Charlie', 35, 'Paris']
]

with open('people.csv', 'w', newline='') as f:
    writer = csv.writer(f)
    writer.writerows(data)  # write multiple rows at once

# Write from list of dictionaries
data_dict = [
    {'name': 'Alice', 'age': 30, 'city': 'New York'},
    {'name': 'Bob', 'age': 25, 'city': 'London'},
]

with open('people_dict.csv', 'w', newline='') as f:
    fieldnames = ['name', 'age', 'city']
    writer = csv.DictWriter(f, fieldnames=fieldnames)
    writer.writeheader()          # writes column names
    writer.writerows(data_dict)   # writes rows
```

### Handling Different Delimiters
CSV can use other delimiters like tab (`\t`) – use the `delimiter` parameter.
```python
reader = csv.reader(f, delimiter='\t')
writer = csv.writer(f, delimiter=';')
```

### Example: Load a Dataset and Compute Statistics
```python
import csv

# Assume we have a CSV file 'students.csv' with columns: name, score
with open('students.csv', 'r') as f:
    reader = csv.DictReader(f)
    scores = []
    for row in reader:
        scores.append(float(row['score']))

average = sum(scores) / len(scores)
print(f"Average score: {average:.2f}")
```

**Why for AI**: Most tabular datasets are in CSV. You'll load training data, split into features and labels, and maybe save predictions back to CSV.

## 4. Realistic AI Examples Combining File I/O, JSON, and CSV

### Example 1: Save Model Training Logs as JSON
Imagine you're training a neural network and want to save the loss and accuracy after each epoch.
```python
import json

# Simulated training logs
logs = []
for epoch in range(1, 6):
    log_entry = {
        "epoch": epoch,
        "loss": round(0.5 / (epoch + 1), 4),
        "accuracy": round(0.7 + epoch * 0.03, 4)
    }
    logs.append(log_entry)

# Save to JSON file
with open('training_logs.json', 'w') as f:
    json.dump(logs, f, indent=4)

# Later, load and analyze
with open('training_logs.json', 'r') as f:
    loaded_logs = json.load(f)
for entry in loaded_logs:
    print(f"Epoch {entry['epoch']}: loss={entry['loss']}, acc={entry['accuracy']}")
```

### Example 2: Load a CSV Dataset and Convert to JSON
Suppose you have a CSV file of Iris flowers and want to convert it to a JSON array for easier handling.
```python
import csv
import json

csv_file = 'iris.csv'   # columns: sepal_length,sepal_width,petal_length,petal_width,species
json_file = 'iris.json'

data = []
with open(csv_file, 'r') as f:
    reader = csv.DictReader(f)
    for row in reader:
        # Convert numeric fields
        row['sepal_length'] = float(row['sepal_length'])
        row['sepal_width'] = float(row['sepal_width'])
        row['petal_length'] = float(row['petal_length'])
        row['petal_width'] = float(row['petal_width'])
        data.append(row)

with open(json_file, 'w') as f:
    json.dump(data, f, indent=4)

print(f"Converted {len(data)} records to {json_file}")
```

### Example 3: Save Model Weights (Simple) – Using JSON
In practice, you'd use NumPy or PyTorch, but here's a toy example saving weights as list of lists.
```python
# Simulated neural network weights
weights = {
    "layer1": [[0.1, 0.2], [0.3, 0.4]],
    "layer2": [0.5, 0.6],
    "bias1": [0.1, 0.1],
    "bias2": [0.2]
}

with open('model_weights.json', 'w') as f:
    json.dump(weights, f, indent=4)

# Load later
with open('model_weights.json', 'r') as f:
    loaded_weights = json.load(f)
```

## 5. Important Notes & Best Practices

- Always use `with` when working with files – it ensures proper cleanup.
- When writing CSV, set `newline=''` to avoid extra blank lines on Windows.
- For large CSV files, consider reading line by line instead of loading all into memory.
- JSON supports only basic data types; you cannot serialize custom objects directly without extra work (but you can convert them to dictionaries).
- For binary data (e.g., images, model weights), use binary modes `'rb'`/`'wb'` and possibly specialized libraries (like `pickle`, `h5py` for models).
