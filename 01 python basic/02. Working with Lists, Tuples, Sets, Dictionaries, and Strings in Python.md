## Lists

A **list** is an ordered, mutable (changeable) collection of items. Items can be of any data type, and you can mix types in a single list.

### Key Points
- Created with square brackets `[]` or the `list()` constructor.
- Indexing starts at `0`. Negative indices count from the end.
- Slicing: `list[start:stop:step]` – `stop` is exclusive.
- Lists are **mutable** – you can change, add, or remove items.

### Common Operations & Methods
| Operation | Example | Description |
|-----------|---------|-------------|
| Append | `my_list.append(4)` | Adds an item to the end |
| Extend | `my_list.extend([5,6])` | Adds multiple items |
| Insert | `my_list.insert(2, 'x')` | Inserts at a specific index |
| Remove | `my_list.remove(3)` | Removes first occurrence of value |
| Pop | `my_list.pop()` or `my_list.pop(1)` | Removes and returns item (last or index) |
| Index | `my_list.index('a')` | Returns index of first occurrence |
| Count | `my_list.count(2)` | Counts occurrences |
| Sort | `my_list.sort()` or `sorted(my_list)` | Sorts the list (in‑place or returns new) |
| Reverse | `my_list.reverse()` | Reverses in‑place |
| Copy | `my_list.copy()` | Returns a shallow copy |
| Clear | `my_list.clear()` | Removes all items |

### Examples
```python
# Creating lists
fruits = ["apple", "banana", "cherry"]
mixed = [1, "hello", 3.14, True]
empty = []

# Indexing and slicing
print(fruits[0])       # apple
print(fruits[-1])      # cherry
print(fruits[1:3])     # ['banana', 'cherry'] (index 1 up to, but not including, 3)

# Modifying
fruits[1] = "blueberry"
print(fruits)          # ['apple', 'blueberry', 'cherry']

# Methods
fruits.append("date")
print(fruits)          # ['apple', 'blueberry', 'cherry', 'date']

fruits.insert(1, "banana")
print(fruits)          # ['apple', 'banana', 'blueberry', 'cherry', 'date']

fruits.remove("cherry")
print(fruits)          # ['apple', 'banana', 'blueberry', 'date']

last = fruits.pop()    # removes and returns 'date'
print(last, fruits)    # date ['apple', 'banana', 'blueberry']

# Looping
for fruit in fruits:
    print(fruit.upper())

# List comprehensions (concise way to create lists)
squares = [x**2 for x in range(5)]          # [0, 1, 4, 9, 16]
even_squares = [x**2 for x in range(10) if x % 2 == 0]  # [0, 4, 16, 36, 64]
```

**Why for AI**: You'll store datasets, sequences of predictions, and training histories in lists. List comprehensions are great for quickly transforming data.

## Tuples

A **tuple** is an ordered, **immutable** collection. Once created, you cannot change, add, or remove items.

### Key Points
- Created with parentheses `()` or the `tuple()` constructor.
- Useful for fixed data that shouldn't change (e.g., coordinates, constant settings).
- Can be unpacked into variables.
- Faster than lists for iteration and access.

### Examples
```python
# Creating tuples
point = (3, 4)
colors = ("red", "green", "blue")
single = (5,)          # trailing comma needed for single-element tuple
empty = ()

# Accessing (same as lists)
print(point[0])        # 3
print(colors[-1])      # blue

# Slicing returns a tuple
print(colors[1:])      # ('green', 'blue')

# Immutability – this would cause an error:
# point[0] = 10        # TypeError

# Unpacking
x, y = point
print(f"x={x}, y={y}") # x=3, y=4

# Useful methods: count, index
print(colors.count("red"))    # 1
print(colors.index("green"))  # 1
```

**Why for AI**: Use tuples for fixed hyperparameters, model configurations, or returning multiple values from a function (like `get_stats` earlier). They're also hashable, so they can be used as dictionary keys (unlike lists).

## Sets

A **set** is an unordered collection of **unique**, immutable elements. Great for membership testing, removing duplicates, and mathematical operations.

### Key Points
- Created with curly braces `{}` or the `set()` constructor (but `{}` creates an empty dictionary, not set – use `set()` for empty set).
- Elements must be immutable (numbers, strings, tuples, but not lists or dictionaries).
- No indexing or slicing because order is not guaranteed.

### Common Operations & Methods
| Operation | Example | Description |
|-----------|---------|-------------|
| Add | `my_set.add(5)` | Adds an element |
| Remove | `my_set.remove(3)` | Removes element (raises KeyError if missing) |
| Discard | `my_set.discard(3)` | Removes if present, no error |
| Pop | `my_set.pop()` | Removes and returns an arbitrary element |
| Clear | `my_set.clear()` | Removes all |
| Union | `set1 \| set2` or `set1.union(set2)` | All elements from both |
| Intersection | `set1 & set2` or `set1.intersection(set2)` | Common elements |
| Difference | `set1 - set2` or `set1.difference(set2)` | Elements in set1 but not set2 |
| Symmetric diff | `set1 ^ set2` or `set1.symmetric_difference(set2)` | Elements in either but not both |
| Subset / Superset | `set1 <= set2`, `set1 >= set2` | Test inclusion |

### Examples
```python
# Creating sets
fruits = {"apple", "banana", "cherry"}
numbers = set([1, 2, 3, 2, 1])   # {1, 2, 3} – duplicates removed
empty = set()

# Adding and removing
fruits.add("orange")
fruits.remove("banana")           # now {'apple', 'cherry', 'orange'}
fruits.discard("kiwi")            # no error

# Set operations
a = {1, 2, 3, 4}
b = {3, 4, 5, 6}

print(a | b)          # {1, 2, 3, 4, 5, 6}
print(a & b)          # {3, 4}
print(a - b)          # {1, 2}
print(a ^ b)          # {1, 2, 5, 6}

# Membership testing is fast
if "apple" in fruits:
    print("Apple is in the set")
```

**Why for AI**: Sets are perfect for storing unique class labels, filtering duplicate data points, or checking if a sample belongs to a category.

## Dictionaries

A **dictionary** stores key-value pairs. Keys must be unique and immutable (strings, numbers, tuples). Values can be any type.

### Key Points
- Created with curly braces `{}` and colons: `{key1: value1, key2: value2}` or `dict()` constructor.
- Access values by key: `my_dict["key"]` or `my_dict.get("key", default)`.
- Dictionaries are **mutable**.

### Common Operations & Methods
| Operation | Example | Description |
|-----------|---------|-------------|
| Access | `value = d["key"]` | Raises KeyError if key missing |
| Get | `d.get("key", 0)` | Returns default if key missing |
| Assign | `d["new_key"] = value` | Adds or updates key-value |
| Delete | `del d["key"]` | Removes key-value pair |
| Pop | `d.pop("key")` | Removes and returns value |
| Keys | `d.keys()` | Returns view of keys |
| Values | `d.values()` | Returns view of values |
| Items | `d.items()` | Returns view of (key, value) tuples |
| Update | `d.update(other_dict)` | Merges another dictionary |
| Clear | `d.clear()` | Removes all items |
| Copy | `d.copy()` | Shallow copy |

### Examples
```python
# Creating dictionaries
person = {
    "name": "Alice",
    "age": 30,
    "city": "New York"
}
empty_dict = {}
grades = dict(math=90, science=85, history=88)  # using keyword arguments

# Accessing
print(person["name"])          # Alice
print(person.get("age"))       # 30
print(person.get("country", "USA"))  # USA (default)

# Modifying
person["age"] = 31
person["job"] = "Engineer"
print(person)
# {'name': 'Alice', 'age': 31, 'city': 'New York', 'job': 'Engineer'}

# Deleting
del person["city"]
age = person.pop("age")        # removes and returns 31

# Looping
for key in person:
    print(key, person[key])

for key, value in person.items():
    print(f"{key}: {value}")

# Checking existence
if "name" in person:
    print("Name exists")

# Dictionary comprehensions
squares = {x: x**2 for x in range(5)}   # {0: 0, 1: 1, 2: 4, 3: 9, 4: 16}
```

**Why for AI**: Dictionaries are everywhere – storing model parameters, mapping labels to indices, holding configuration settings, representing a single data sample (like a row with feature names), and handling JSON data.

## String Operations

Strings are sequences of characters. They are **immutable** – any operation that seems to modify a string actually creates a new one.

### Key Points
- Created with single `' '` or double `" "` quotes.
- Triple quotes `""" """` for multi-line strings.
- Indexing and slicing work just like lists.

### Common String Methods
| Method | Example | Description |
|--------|---------|-------------|
| `lower()` | `"Hello".lower()` → `"hello"` | Converts to lowercase |
| `upper()` | `"Hello".upper()` → `"HELLO"` | Converts to uppercase |
| `strip()` | `"  hi  ".strip()` → `"hi"` | Removes leading/trailing whitespace |
| `split()` | `"a,b,c".split(",")` → `['a','b','c']` | Splits into list |
| `join()` | `"-".join(['a','b','c'])` → `"a-b-c"` | Joins list into string |
| `replace()` | `"I like cats".replace("cats","dogs")` → `"I like dogs"` | Replaces substring |
| `find()` | `"hello".find("e")` → `1` | Returns index of first occurrence (-1 if not found) |
| `startswith()` | `"hello".startswith("he")` → `True` | Checks prefix |
| `endswith()` | `"hello".endswith("lo")` → `True` | Checks suffix |
| `count()` | `"abracadabra".count("a")` → `5` | Counts occurrences |
| `isalpha()`, `isdigit()`, `isalnum()` | Checks character types |

### String Formatting
- **f-strings** (Python 3.6+): `f"Hello, {name}!"` – easiest and most readable.
- **`.format()`**: `"Hello, {}".format(name)`.
- **`%` formatting**: `"Hello, %s" % name` (older style).

### Examples
```python
text = "  Artificial Intelligence  "

# Basic operations
print(text.lower())          # "  artificial intelligence  "
print(text.upper())          # "  ARTIFICIAL INTELLIGENCE  "
print(text.strip())          # "Artificial Intelligence"
print(text.replace("Intelligence", "Intelligence (AI)"))
# "  Artificial Intelligence (AI)  "

# Splitting and joining
sentence = "Python is great for AI"
words = sentence.split()     # ['Python', 'is', 'great', 'for', 'AI']
back_together = " ".join(words)  # "Python is great for AI"

csv_line = "apple,banana,cherry"
items = csv_line.split(",")  # ['apple', 'banana', 'cherry']

# Searching
idx = sentence.find("great")  # 10
print(idx)
print("great" in sentence)    # True

# Formatting
name = "Alex"
score = 98.5
print(f"{name} scored {score}%")          # Alex scored 98.5%
print("{} scored {}%".format(name, score)) # same
print("%s scored %.1f%%" % (name, score))  # older style (%% to print %)

# Multi-line string
message = """
Dear Student,
Keep learning Python!
It will pay off.
"""
print(message)
```

**Why for AI**: Text data is everywhere – you'll preprocess text by cleaning, splitting, lowercasing, etc. Strings are also used for reading/writing files, handling paths, and logging.

## Putting It All Together – A Realistic AI Example

Imagine you're building a simple text classifier. You need to:
- Store training data (text + label)
- Remove duplicate samples (using set on text)
- Build a vocabulary (dictionary mapping words to indices)
- Preprocess each text (split, lowercase, strip)

Let's simulate this small pipeline:

```python
# Raw data: list of (text, label) tuples
raw_data = [
    ("I love this movie", "positive"),
    ("This film is terrible", "negative"),
    ("Awesome movie, loved it", "positive"),
    ("I love this movie", "positive"),   # duplicate
    ("Boring and dull", "negative"),
    ("Not bad, actually good", "positive"),
]

# 1. Remove duplicates based on text using a set
unique_texts = set()
cleaned_data = []
for text, label in raw_data:
    if text not in unique_texts:
        unique_texts.add(text)
        cleaned_data.append((text, label))

print("Cleaned data (no duplicate texts):")
for item in cleaned_data:
    print(item)

# 2. Build vocabulary: map each unique word to an index
vocab = {}
index = 0
for text, _ in cleaned_data:
    words = text.lower().split()  # simple tokenization
    for word in words:
        if word not in vocab:
            vocab[word] = index
            index += 1

print("\nVocabulary:")
print(vocab)

# 3. Preprocess a new sample into a bag-of-words vector (list of counts)
new_text = "I loved this awesome movie"
words = new_text.lower().split()
vector = [0] * len(vocab)   # initialize list of zeros
for word in words:
    if word in vocab:
        vector[vocab[word]] += 1

print(f"\nVector for '{new_text}':")
print(vector)

# 4. Store labels as a set to see all possible classes
labels = set(label for _, label in cleaned_data)
print(f"\nUnique labels: {labels}")
```

This example uses tuples, lists, sets, dictionaries, and string operations together – exactly the kind of data wrangling you'll do in AI.
