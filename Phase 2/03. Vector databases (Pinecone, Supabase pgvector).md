# 📘 Vector Databases: Pinecone & Supabase pgvector — Complete Guide

## 1. What Are Vector Databases?

A **vector database** is a specialized database designed to store, manage, and query high-dimensional vector embeddings efficiently. Unlike traditional databases that excel at exact matches and structured queries, vector databases are built for **similarity search**—finding data points that are conceptually similar to a query vector.

### Why Vector Databases Matter

When you convert text (or images, audio, etc.) into embeddings, you get arrays of hundreds or thousands of numbers. Traditional databases can't efficiently search for "similar" vectors because:
- They lack native support for vector similarity operations
- They can't leverage specialized indexing algorithms
- Performance degrades dramatically at scale

Vector databases solve these problems with:
- **Specialized indexing** (HNSW, IVF) for fast approximate nearest neighbor search
- **Built-in similarity metrics** (cosine, Euclidean, dot product)
- **Hybrid search** combining vector similarity with metadata filtering
- **Scale to billions** of vectors with low latency

## 2. Two Approaches: Pinecone vs. Supabase pgvector

Before diving deep, understand the fundamental difference:

| Aspect | **Pinecone** | **Supabase pgvector** |
|--------|--------------|------------------------|
| **Type** | Fully managed vector database service | PostgreSQL extension for vectors |
| **Hosting** | Pinecone cloud (AWS/GCP) | Self-hosted or Supabase platform |
| **Learning Curve** | Low—simple API, minimal setup | Moderate—requires SQL knowledge |
| **Integration** | Works with any app via REST/SDK | Lives inside your PostgreSQL database |
| **Pricing** | Pay per usage (index size + operations) | Part of Supabase free tier or hosting costs |
| **Best For** | Teams wanting zero ops, quick start | Teams already using Postgres, need data relationships |

## 3. Pinecone: Fully Managed Vector Database

### 3.1 Core Concepts

**Index** – The top-level container for vectors, analogous to a table in SQL. Each index has:
- **Dimension**: Size of vectors it stores (e.g., 1536 for OpenAI embeddings)
- **Metric**: Similarity measure (cosine, euclidean, dotproduct)
- **Cloud/Region**: Where your index is hosted

**Namespace** – A way to partition vectors within an index (like folders). Useful for multi-tenancy or separating data types.

**Metadata** – Key-value pairs attached to each vector for filtering and organizing data.

### 3.2 Getting Started with Pinecone

#### Step 1: Create an Account
Sign up at [pinecone.io](https://www.pinecone.io) – free starter plan available.

#### Step 2: Get API Key
From your dashboard, create an API key and note your environment/region.

#### Step 4: Initialize Client
```javascript
import { Pinecone } from '@pinecone-database/pinecone';

const pc = new Pinecone({
  apiKey: process.env.PINECONE_API_KEY
});
```

### 3.3 Creating an Index

Pinecone offers two index types:

**Option A: Serverless Index (Recommended)**
```javascript
await pc.createIndex({
  name: 'my-documents',
  dimension: 1536,              // Match your embedding model
  metric: 'cosine',              // Similarity metric
  spec: { 
    serverless: { 
      cloud: 'aws', 
      region: 'us-east-1' 
    }
  }
});
```

**Option B: Integrated Embedding Index**
For simpler workflows, let Pinecone handle embeddings:
```javascript
await pc.createIndexForModel({
  name: 'my-docs',
  cloud: 'aws',
  region: 'us-east-1',
  embed: {
    model: 'llama-text-embed-v2',  // Pinecone's embedding model
    fieldMap: { text: 'content' }   // Which field to embed
  }
});
```

### 3.4 Inserting Vectors

**With pre-generated embeddings:**
```javascript
const index = pc.index('my-documents');

const vectors = [{
  id: 'doc1',
  values: [0.023, -0.045, ...],  // Your embedding array
  metadata: {
    title: 'AI Introduction',
    category: 'technology',
    url: 'https://example.com/doc1'
  }
}];

await index.upsert(vectors);
```

**With integrated embeddings (simpler!)** :
```javascript
const denseIndex = pc.Index('my-docs');

const records = [{
  _id: 'rec1',
  content: 'The Eiffel Tower was completed in 1889...',
  category: 'history'
}];

await denseIndex.upsertRecords('my-namespace', records);
// Pinecone automatically generates embeddings!
```

### 3.5 Querying for Similar Vectors

**Basic semantic search**:
```javascript
const results = await denseIndex.search({
  namespace: 'my-namespace',
  query: {
    topK: 10,                     // Return 10 most similar
    inputs: { text: 'Famous historical landmarks' }
  }
});

for (const hit of results.result.hits) {
  console.log(`Score: ${hit._score}, Text: ${hit.fields.content}`);
}
```

**With metadata filtering**:
```javascript
const filtered = await denseIndex.search({
  namespace: 'my-namespace',
  query: { topK: 10, inputs: { text: query } },
  filter: { 
    category: { "$eq": "history" },
    year: { "$gte": 1800 }
  }
});
```

**With reranking for better relevance**:
```javascript
const reranked = await denseIndex.search({
  namespace: 'my-namespace',
  query: { topK: 20, inputs: { text: query } }, // Get more candidates
  rerank: {
    model: 'bge-reranker-v2-m3',   // Specialized reranking model
    topN: 10,                       // Return top 10 after reranking
    rankFields: ['content']          // Which field to evaluate
  }
});
```

### 3.6 Best Practices for Pinecone

- **Batch uploads**: Insert records in batches of 100–1000 for efficiency
- **Use namespaces**: Isolate different datasets or tenants
- **Monitor stats**: `describeIndexStats()` to track vector count and storage
- **Enable deletion protection** for production indexes
- **Cache results** for frequently asked queries to reduce costs

## 4. Supabase pgvector: Vector Search in PostgreSQL

### 4.1 What is pgvector?

**pgvector** is an open-source extension that adds vector similarity search to PostgreSQL. With Supabase, you get:
- A full PostgreSQL database
- The pgvector extension pre-enabled (or easily enabled)
- Integration with your existing data and tables
- Supabase's auth, storage, and realtime features

### 4.2 Core Concepts

**Vector column** – A column in any table with type `vector(n)` where n is dimensions.

**Similarity operators**:
- `<->` Euclidean distance (L2)
- `<#>` Negative inner product
- `<=>` Cosine similarity

**Indexes** – Special indexes (IVFFlat, HNSW) for fast similarity search.

### 4.3 Setting Up pgvector in Supabase

#### Step 1: Enable the Extension
Via SQL Editor:
```sql
-- Enable the vector extension
create extension if not exists vector
with schema extensions;
```

Or via Dashboard: Database → Extensions → Enable "vector".

#### Step 2: Create a Table with Vector Column
```sql
create table documents (
  id bigserial primary key,
  title text not null,
  content text not null,
  url text,
  category text,
  embedding vector(1536)  -- Store OpenAI embeddings here
);
```

#### Step 3: Create a Search Function
```sql
create function match_documents (
  query_embedding vector(1536),
  match_count int default 10,
  filter_category text default null
) returns table (
  id bigint,
  content text,
  similarity float
)
language plpgsql
as $$
begin
  return query
  select
    documents.id,
    documents.content,
    1 - (documents.embedding <=> query_embedding) as similarity
  from documents
  where 
    (filter_category is null or documents.category = filter_category)
  order by documents.embedding <=> query_embedding
  limit match_count;
end;
$$;
```

This function uses the `<=>` cosine distance operator and converts it to similarity.

### 4.4 Generating and Storing Embeddings

#### Option A: Generate in App, Store via Supabase JS
```javascript
import { createClient } from '@supabase/supabase-js';
import OpenAI from 'openai';

const supabase = createClient(process.env.SUPABASE_URL, process.env.SUPABASE_KEY);
const openai = new OpenAI({ apiKey: process.env.OPENAI_API_KEY });

async function addDocument(title, content) {
  // Generate embedding
  const response = await openai.embeddings.create({
    model: 'text-embedding-3-small',
    input: content
  });
  const embedding = response.data[0].embedding;

  // Store in Supabase
  const { data, error } = await supabase
    .from('documents')
    .insert({
      title,
      content,
      embedding
    });
  
  return data;
}
```

#### Option B: Automatic Embeddings with Edge Functions

Supabase offers a production-ready pattern using triggers, queues, and Edge Functions:

```sql
-- 1. Enable required extensions
create extension if not exists pgmq;      -- Queue
create extension if not exists pg_net;    -- Async HTTP
create extension if not exists pg_cron;   -- Scheduling

-- 2. Create queue for embedding jobs
select pgmq.create('embedding_jobs');

-- 3. Create trigger function to queue embedding jobs
create or replace function queue_embeddings()
returns trigger
language plpgsql
as $$
begin
  perform pgmq.send(
    'embedding_jobs',
    jsonb_build_object(
      'id', NEW.id,
      'content', NEW.content,
      'table', 'documents'
    )
  );
  return NEW;
end;
$$;

-- 4. Create trigger on insert/update
create trigger documents_embed_trigger
  after insert or update of content on documents
  for each row
  execute function queue_embeddings();

-- 5. Schedule processing every 10 seconds
select cron.schedule(
  'process-embeddings',
  '10 seconds',
  $$select process_embedding_queue();$$
);
```

The Edge Function would then pick up jobs, generate embeddings via OpenAI, and update the records.

### 4.5 Querying Similar Documents

**Using the match function**:
```javascript
async function searchSimilar(query, category = null) {
  // Generate embedding for query
  const response = await openai.embeddings.create({
    model: 'text-embedding-3-small',
    input: query
  });
  const queryEmbedding = response.data[0].embedding;

  // Call the match function
  const { data, error } = await supabase
    .rpc('match_documents', {
      query_embedding: queryEmbedding,
      match_count: 10,
      filter_category: category
    });

  return data;
}
```

**Direct SQL with filtering**:
```sql
select 
  id,
  content,
  1 - (embedding <=> '[0.023, -0.045, ...]') as similarity
from documents
where category = 'history'
order by embedding <=> '[0.023, -0.045, ...]'
limit 10;
```

### 4.6 Advanced: Hybrid Search with Full-Text

Combine vector similarity with PostgreSQL's built-in full-text search:

```sql
create function hybrid_search(
  query_text text,
  query_embedding vector(1536),
  match_count int default 10
) returns table (
  id bigint,
  content text,
  vector_score float,
  text_score float,
  combined_score float
)
language plpgsql
as $$
begin
  return query
  select
    id,
    content,
    1 - (embedding <=> query_embedding) as vector_score,
    ts_rank(to_tsvector('english', content), plainto_tsquery('english', query_text)) as text_score,
    (0.7 * (1 - (embedding <=> query_embedding)) + 
     0.3 * ts_rank(to_tsvector('english', content), plainto_tsquery('english', query_text))) as combined_score
  from documents
  where to_tsvector('english', content) @@ plainto_tsquery('english', query_text)
     or true  -- include all for vector-only results
  order by combined_score desc
  limit match_count;
end;
$$;
```

## 5. Building a RAG Application: Platform Comparison

### 5.1 Complete RAG with Pinecone

```javascript
// ingest.js - Load documents into Pinecone
import { Pinecone } from '@pinecone-database/pinecone';
import { OpenAIEmbeddings } from '@langchain/openai';
import { RecursiveCharacterTextSplitter } from 'langchain/text_splitter';
import { PineconeStore } from '@langchain/pinecone';

async function ingestDocument(filePath) {
  // Load and split document
  const loader = new TextLoader(filePath);
  const docs = await loader.load();
  
  const splitter = new RecursiveCharacterTextSplitter({
    chunkSize: 1000,
    chunkOverlap: 200
  });
  const chunks = await splitter.splitDocuments(docs);

  // Generate embeddings and store
  const embeddings = new OpenAIEmbeddings();
  const pinecone = new Pinecone({ apiKey: process.env.PINECONE_API_KEY });
  const index = pinecone.Index(process.env.PINECONE_INDEX);
  
  await PineconeStore.fromDocuments(chunks, embeddings, {
    pineconeIndex: index,
    namespace: 'my-docs'
  });
}

// query.js - Ask questions
async function ask(question) {
  const embeddings = new OpenAIEmbeddings();
  const pinecone = new Pinecone({ apiKey: process.env.PINECONE_API_KEY });
  const index = pinecone.Index(process.env.PINECONE_INDEX);
  
  const vectorStore = await PineconeStore.fromExistingIndex(embeddings, {
    pineconeIndex: index,
    namespace: 'my-docs'
  });
  
  const retriever = vectorStore.asRetriever();
  const docs = await retriever.getRelevantDocuments(question);
  
  // Use docs with LLM to generate answer
  return generateAnswer(question, docs);
}
```

### 5.2 Complete RAG with Supabase pgvector

```sql
-- First, create the documents table
create table documents (
  id bigserial primary key,
  content text,
  metadata jsonb,
  embedding vector(1536)
);

-- Create match function
create function match_documents (
  query_embedding vector(1536),
  match_count int default 10,
  filter jsonb default '{}'
) returns table (
  id bigint,
  content text,
  metadata jsonb,
  similarity float
)
language plpgsql
as $$
begin
  return query
  select
    id,
    content,
    metadata,
    1 - (documents.embedding <=> query_embedding) as similarity
  from documents
  where metadata @> filter
  order by documents.embedding <=> query_embedding
  limit match_count;
end;
$$;
```

```javascript
// app.js - Node.js implementation
import { createClient } from '@supabase/supabase-js';
import OpenAI from 'openai';

const supabase = createClient(process.env.SUPABASE_URL, process.env.SUPABASE_KEY);
const openai = new OpenAI({ apiKey: process.env.OPENAI_API_KEY });

async function addDocument(content, metadata) {
  // Generate embedding
  const embedding = await openai.embeddings.create({
    model: 'text-embedding-3-small',
    input: content
  });

  // Store in Supabase
  const { data, error } = await supabase
    .from('documents')
    .insert({
      content,
      metadata,
      embedding: embedding.data[0].embedding
    });
  
  return data;
}

async function searchAndAnswer(question, filter = {}) {
  // Embed question
  const embedding = await openai.embeddings.create({
    model: 'text-embedding-3-small',
    input: question
  });

  // Search similar documents
  const { data: docs, error } = await supabase
    .rpc('match_documents', {
      query_embedding: embedding.data[0].embedding,
      match_count: 5,
      filter
    });

  // Generate answer with context
  const context = docs.map(d => d.content).join('\n\n');
  const answer = await generateLLMResponse(question, context);
  
  return { answer, sources: docs };
}
```

## 6. Platform Comparison: Choosing the Right Tool

| Factor | Pinecone | Supabase pgvector |
|--------|----------|-------------------|
| **Setup Time** | Minutes (cloud service) | Minutes (if using Supabase) |
| **Operations** | Zero – fully managed | You manage scaling, backups |
| **Cost Model** | Pay per vector index | Database hosting + API costs |
| **Query Speed** | Optimized, consistent | Depends on indexing |
| **Metadata Filtering** | Rich query language | PostgreSQL JSONB operators |
| **Hybrid Search** | Via sparse-dense or reranking | Combine with full-text search |
| **Integration** | Any app via REST/SDK | Native to Postgres ecosystem |
| **Multi-tenancy** | Namespaces | Separate tables or row-level security |
| **Free Tier** | 1 index, limited vectors | Generous with Supabase free plan |

### When to Choose Pinecone
- You want zero ops and fast implementation
- You're building a pure vector search application
- You don't already use PostgreSQL
- You need advanced features like integrated embeddings or reranking

### When to Choose Supabase pgvector
- You already use Supabase/PostgreSQL
- Your data has complex relationships beyond vectors
- You need Row Level Security, realtime, or other Postgres features
- You want to keep everything in one database

## 7. Summary: Key Takeaways

1. **Vector databases are essential** for semantic search, RAG, and recommendation systems.

2. **Pinecone** offers a fully managed, developer-friendly experience with features like integrated embeddings and reranking.

3. **Supabase pgvector** brings vector search to PostgreSQL, letting you leverage your existing data and SQL skills.

4. **Both support**:
   - Cosine, Euclidean, and dot product similarity
   - Metadata filtering
   - High-performance indexing
   - Integration with LangChain and OpenAI

5. **Your choice depends on**:
   - Existing infrastructure
   - Operations team/capacity
   - Need for complex relational queries
   - Budget and scale requirements
