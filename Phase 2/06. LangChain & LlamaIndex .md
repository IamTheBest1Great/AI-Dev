# 📘 LangChain.js / LlamaIndex.ts Basics — Complete Guide

After building a RAG pipeline from scratch, you'll appreciate **orchestration frameworks** that abstract away repetitive tasks. LangChain.js and LlamaIndex.ts are the two most popular libraries for building AI applications in JavaScript/TypeScript. This guide covers the fundamentals of both, so you can choose the right tool for your project.

---

## 1. Why Use an Orchestration Framework?

Building AI applications involves many moving parts:
- Calling LLMs (with prompts, history, streaming)
- Loading and splitting documents
- Generating embeddings
- Storing and retrieving vectors
- Chaining multiple steps (retrieve → generate)
- Giving the LLM tools/function calling

Frameworks like LangChain and LlamaIndex provide **abstractions** and **pre‑built components** that glue these pieces together, so you can focus on your application logic rather than reinventing the wheel.

---

## 2. LangChain.js Basics

LangChain is a comprehensive framework with a modular design. It's great for building **chains**, **agents**, and complex workflows.

### 2.1 Installation

```bash
npm install langchain @langchain/openai @langchain/pinecone @langchain/community
```

### 2.2 Core Concepts

| Concept | Description |
|---------|-------------|
| **Model** | LLM wrapper (ChatOpenAI, OpenAIEmbeddings) |
| **Prompt** | Template for sending instructions to the LLM |
| **Chain** | Sequence of calls (e.g., prompt → model → output parser) |
| **Document** | A piece of text with metadata |
| **Document Loader** | Loads documents from various sources (PDF, CSV, web) |
| **Text Splitter** | Splits documents into chunks |
| **Vector Store** | Wrapper around vector databases (Pinecone, Supabase) |
| **Retriever** | Interface for fetching relevant documents from a vector store |
| **Tool** | A function the LLM can call (e.g., calculator, API) |
| **Agent** | LLM that decides which tools to use and in what order |

### 2.3 Example 1: Simple LLM Call

```typescript
import { ChatOpenAI } from '@langchain/openai';
import { HumanMessage } from '@langchain/core/messages';

const model = new ChatOpenAI({ model: 'gpt-4o-mini', temperature: 0 });
const response = await model.invoke([new HumanMessage('Tell me a joke.')]);
console.log(response.content);
```

### 2.4 Example 2: RAG Pipeline with LangChain

This example loads a PDF, splits it, embeds the chunks, stores them in Pinecone, and answers a question.

```typescript
import { PDFLoader } from 'langchain/document_loaders/fs/pdf';
import { RecursiveCharacterTextSplitter } from 'langchain/text_splitter';
import { OpenAIEmbeddings } from '@langchain/openai';
import { PineconeStore } from '@langchain/pinecone';
import { Pinecone } from '@pinecone-database/pinecone';
import { ChatOpenAI } from '@langchain/openai';
import { createRetrievalChain } from 'langchain/chains/retrieval';
import { createStuffDocumentsChain } from 'langchain/chains/combine_documents';
import { ChatPromptTemplate } from '@langchain/core/prompts';

// 1. Load document
const loader = new PDFLoader('./document.pdf');
const docs = await loader.load();

// 2. Split into chunks
const splitter = new RecursiveCharacterTextSplitter({
  chunkSize: 1000,
  chunkOverlap: 200,
});
const splitDocs = await splitter.splitDocuments(docs);

// 3. Create embeddings and vector store
const embeddings = new OpenAIEmbeddings({ model: 'text-embedding-3-small' });
const pc = new Pinecone({ apiKey: process.env.PINECONE_API_KEY });
const index = pc.index(process.env.PINECONE_INDEX);
const vectorStore = await PineconeStore.fromDocuments(splitDocs, embeddings, { pineconeIndex: index });

// 4. Build retriever
const retriever = vectorStore.asRetriever({ k: 5 });

// 5. Create a chain that combines retrieved documents
const llm = new ChatOpenAI({ model: 'gpt-4o-mini', temperature: 0 });
const prompt = ChatPromptTemplate.fromTemplate(`
Answer the question based only on the following context:
{context}

Question: {input}
`);
const documentChain = await createStuffDocumentsChain({ llm, prompt });
const retrievalChain = await createRetrievalChain({ retriever, combineDocsChain: documentChain });

// 6. Ask a question
const result = await retrievalChain.invoke({ input: 'What is the main topic?' });
console.log(result.answer);
```

### 2.5 Tools and Agents (Brief)

LangChain also supports agents that can use tools. For example:

```typescript
import { createReactAgent } from '@langchain/langgraph/prebuilt';
import { TavilySearchResults } from '@langchain/community/tools/tavily_search';

const tools = [new TavilySearchResults({ maxResults: 1 })];
const agent = await createReactAgent({ llm, tools });
const result = await agent.invoke({ messages: [{ role: 'user', content: 'What's the weather in Paris?' }] });
```

## 3. LlamaIndex.ts Basics

LlamaIndex (formerly GPT Index) focuses specifically on **data indexing and retrieval**. It's designed to make RAG extremely easy, with a higher‑level abstraction than LangChain.

### 3.1 Installation

```bash
npm install llamaindex @llamaindex/openai @llamaindex/pinecone
```

### 3.2 Core Concepts

| Concept | Description |
|---------|-------------|
| **Document** | A container for text and metadata |
| **Node** | A chunk of a document (similar to LangChain's Document after splitting) |
| **Embedding** | Generated by an embedding model |
| **VectorStore** | Interface to store nodes and their embeddings |
| **Index** | A data structure that organizes nodes for efficient retrieval |
| **QueryEngine** | An interface to ask questions over an index |
| **Retriever** | Fetches nodes based on a query |
| **ResponseSynthesizer** | Combines retrieved nodes into an answer using an LLM |

### 3.3 Example: RAG Pipeline with LlamaIndex.ts

```typescript
import {
  Document,
  VectorStoreIndex,
  SimpleDirectoryReader,
  OpenAIEmbedding,
  OpenAI,
  Settings,
} from 'llamaindex';
import { PineconeVectorStore } from '@llamaindex/pinecone';
import { Pinecone } from '@pinecone-database/pinecone';

// Configure global settings (optional)
Settings.llm = new OpenAI({ model: 'gpt-4o-mini', temperature: 0 });
Settings.embedModel = new OpenAIEmbedding({ model: 'text-embedding-3-small' });

// 1. Load document (you can also use SimpleDirectoryReader for files)
const document = new Document({ text: '...', metadata: { source: 'file.pdf' } });

// 2. Split into nodes (automatic with default transformations)
// You can configure the splitter globally or per index.

// 3. Set up vector store (Pinecone)
const pc = new Pinecone({ apiKey: process.env.PINECONE_API_KEY });
const pineconeIndex = pc.index(process.env.PINECONE_INDEX);
const vectorStore = new PineconeVectorStore({ pineconeIndex });

// 4. Create index (this will embed and store nodes)
const index = await VectorStoreIndex.fromDocuments([document], {
  vectorStore,
});

// 5. Create query engine
const queryEngine = index.asQueryEngine({
  retriever: { similarityTopK: 5 }, // retrieve top 5
});

// 6. Query
const response = await queryEngine.query({ query: 'What is the main topic?' });
console.log(response.toString());
```

LlamaIndex automatically handles chunking, embedding, and retrieval with sensible defaults.

### 3.4 Advanced: Custom Chunking

You can customize the node parser:

```typescript
import { SentenceSplitter } from 'llamaindex';

const splitter = new SentenceSplitter({ chunkSize: 1024, chunkOverlap: 200 });
const nodes = splitter.splitDocuments([document]);
```

Then build the index from nodes directly.

## 4. LangChain vs. LlamaIndex: Comparison

| Aspect | LangChain | LlamaIndex |
|--------|-----------|------------|
| **Philosophy** | Swiss Army knife for all things LLM – chains, agents, tools, memory | Focused on data indexing and retrieval (RAG) |
| **Abstraction Level** | Lower; you compose components explicitly | Higher; many defaults, less code for RAG |
| **Flexibility** | Extremely flexible – you can build any workflow | Flexible but geared toward retrieval |
| **Learning Curve** | Steeper; many concepts | Gentler; fewer concepts for RAG |
| **Agents & Tools** | Excellent built‑in support | Agents exist but are less mature |
| **Integration Ecosystem** | Huge; many connectors (databases, APIs, vector stores) | Good; growing, especially for vector stores |
| **When to Use** | Need complex chains, agents, or tool‑calling | Primarily building RAG applications; want quick setup |

**Rule of thumb**:
- Use **LangChain** when you need an **agent** (tool‑calling), complex multi‑step workflows, or you're already comfortable with its concepts.
- Use **LlamaIndex** when your primary goal is **RAG** (querying over documents) and you want less boilerplate.

Many projects use both: LlamaIndex for indexing/retrieval and LangChain for the agent layer.

## 5. Best Practices and Tips

- **Always set a `temperature=0` for factual Q&A** to reduce hallucinations.
- **Experiment with chunk sizes** – too small loses context, too large reduces retrieval precision.
- **Use metadata filtering** to improve retrieval when you have many documents.
- **Cache embeddings** for frequently used queries to reduce cost.
- **Monitor costs** – frameworks make it easy to call models, but each call costs money.
- **Keep up with updates** – both frameworks evolve rapidly; always check the latest docs.
