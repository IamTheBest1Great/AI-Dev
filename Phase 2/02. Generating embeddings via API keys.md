# 📘 Generating Embeddings via API — Complete Guide

## 1. What Does "Generating Embeddings via API" Mean?

Instead of training your own embedding model or running one locally, you use a **cloud API** to convert your text into vectors. You send text to an endpoint, and it returns the numerical representation (embeddings) ready for use in your application .

### Why Use an API?
- **No infrastructure** – No need to manage GPUs or model hosting
- **State‑of‑the‑art models** – Access the latest models from OpenAI, Cohere, etc.
- **Scale on demand** – Handle from one to millions of requests
- **Focus on your app** – Let the API provider handle the ML complexity

### Step 3: Make Your First Embedding Call

#### **OpenAI Example** (JavaScript)
```javascript
import OpenAI from 'openai';

const openai = new OpenAI({
  apiKey: process.env.OPENAI_API_KEY
});

async function getEmbedding(text) {
  const response = await openai.embeddings.create({
    model: "text-embedding-3-small",
    input: text,
    encoding_format: "float"  // Can also use "base64"
  });
  
  return response.data[0].embedding; // Array of 1536 numbers
}

// Usage
const embedding = await getEmbedding("What is the capital of France?");
console.log(embedding.length); // 1536
console.log(embedding.slice(0, 5)); // First 5 numbers: [0.023, -0.045, 0.012, ...]
```

#### **Cohere Example** (JavaScript)
```javascript
import { CohereClient } from 'cohere-ai';

const cohere = new CohereClient({
  token: process.env.COHERE_API_KEY
});

async function getCohereEmbedding(text) {
  const response = await cohere.embed({
    texts: [text],
    model: "embed-english-v3.0",
    inputType: "search_query"  // or "search_document" for docs
  });
  
  return response.embeddings[0]; // Array of 1024 numbers
}
```

## 4. Advanced Usage — Batching for Efficiency

APIs accept **multiple texts in one call**, which is faster and cheaper .

```javascript
async function getEmbeddingsBatch(texts) {
  // OpenAI supports up to 2048 texts per call
  const response = await openai.embeddings.create({
    model: "text-embedding-3-small",
    input: texts,  // Array of strings
  });
  
  // Returns embeddings in same order as input
  return response.data.map(item => item.embedding);
}

// Example: embed 10 documents at once
const documents = [
  "Document 1 text...",
  "Document 2 text...",
  // ... up to 2048
];
const embeddings = await getEmbeddingsBatch(documents);
```

**Benefits of batching** :
- **Lower latency** – One network call instead of many
- **Lower cost** – Fewer API requests
- **Rate limit friendly** – Stays within requests/minute limits

## 5. Handling Large Documents — Chunking Strategy

Embedding models have **token limits** (e.g., 8,192 tokens for OpenAI's `text-embedding-3-small`). For longer documents, you must **chunk** them first .

```javascript
import { encode } from 'gpt-tokenizer'; // or tiktoken

function chunkText(text, maxTokens = 8000, overlap = 200) {
  const tokens = encode(text);
  const chunks = [];
  
  for (let i = 0; i < tokens.length; i += maxTokens - overlap) {
    const chunkTokens = tokens.slice(i, i + maxTokens);
    chunks.push(decode(chunkTokens)); // Convert back to text
  }
  
  return chunks;
}

// Usage
const longDoc = "Very long document text..."; // 20,000 tokens
const chunks = chunkText(longDoc, 8000, 200);

// Embed each chunk
const chunkEmbeddings = await Promise.all(
  chunks.map(chunk => getEmbedding(chunk))
);
```

**Best practices** :
- **Chunk size**: 500–1000 tokens for most RAG applications
- **Overlap**: 10–20% to maintain context between chunks
- **Strategy**: Split by paragraphs, sentences, or fixed token count

## 6. Error Handling and Retries

APIs can fail — network issues, rate limits, server errors. Implement robust error handling .

```javascript
async function getEmbeddingWithRetry(text, maxRetries = 3) {
  for (let i = 0; i < maxRetries; i++) {
    try {
      return await getEmbedding(text);
    } catch (error) {
      if (i === maxRetries - 1) throw error;
      
      // Rate limit hit? Wait and retry
      if (error.status === 429) {
        const waitTime = (i + 1) * 1000; // Exponential backoff
        await new Promise(resolve => setTimeout(resolve, waitTime));
      } else {
        // Other errors, maybe retry immediately
        await new Promise(resolve => setTimeout(resolve, 1000));
      }
    }
  }
}
```

**Common error codes**:
- `429` – Too many requests (rate limited)
- `500` / `503` – Server error (retry with backoff)
- `401` – Invalid API key
- `400` – Bad request (check input format)

## 7. Cost Optimization Strategies

### 1. **Cache embeddings** for frequently accessed text
```javascript
const embeddingCache = new Map();

async function getEmbeddingCached(text) {
  if (embeddingCache.has(text)) {
    return embeddingCache.get(text);
  }
  const embedding = await getEmbedding(text);
  embeddingCache.set(text, embedding);
  return embedding;
}
```

### 2. **Use smaller models** when possible
- `text-embedding-3-small` is 1/5 the cost of `text-embedding-3-large`
- Test if small model meets your accuracy needs

### 3. **Batch strategically** – Don't batch if you only need one embedding occasionally
- For single requests, just call once
- For bulk processing, batch as much as possible 

### 4. **Compress dimensions** if your use case allows
- OpenAI supports `dimensions` parameter to truncate vectors
- Smaller vectors = cheaper storage and faster queries

```javascript
const response = await openai.embeddings.create({
  model: "text-embedding-3-small",
  input: text,
  dimensions: 256  // Truncate to 256 dimensions
});
```

## 8. Complete Workflow Example: Text → Embeddings → Store

```javascript
import OpenAI from 'openai';
import { createClient } from '@supabase/supabase-js';

const openai = new OpenAI({ apiKey: process.env.OPENAI_API_KEY });
const supabase = createClient(process.env.SUPABASE_URL, process.env.SUPABASE_KEY);

async function processDocument(document) {
  // 1. Split into chunks
  const chunks = chunkText(document.content, 8000, 200);
  
  // 2. Generate embeddings for all chunks in batch
  const embeddings = await getEmbeddingsBatch(chunks);
  
  // 3. Store in database (Supabase pgvector)
  const records = chunks.map((chunk, i) => ({
    document_id: document.id,
    content: chunk,
    embedding: embeddings[i],
    chunk_index: i
  }));
  
  const { error } = await supabase
    .from('document_chunks')
    .insert(records);
    
  if (error) console.error('Error storing embeddings:', error);
  return records.length;
}

// Helper function
async function getEmbeddingsBatch(texts) {
  const response = await openai.embeddings.create({
    model: "text-embedding-3-small",
    input: texts,
  });
  return response.data.map(item => item.embedding);
}
```

## 10. Common Pitfalls and Solutions

| Pitfall | Solution |
|---------|----------|
| **Token limit exceeded** | Chunk your text before embedding  |
| **Rate limited** | Implement exponential backoff, batch requests  |
| **Costs too high** | Cache results, use smaller model, batch efficiently |
| **Poor search results** | Check chunk size/strategy, use better model, add metadata filtering |
| **Slow performance** | Batch requests, use async/concurrent calls |
| **API key exposed** | Use environment variables, never commit keys to git |
