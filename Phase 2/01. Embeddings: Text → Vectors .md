# 📘 Word Embeddings: Text → Vectors — Complete Guide

## 1. What Are Embeddings?

**Embeddings** are numerical representations of text (words, sentences, paragraphs) as **vectors** — lists of floating‑point numbers — in a high‑dimensional space.  

Think of them as **coordinates** that place each piece of text in a “meaning space.” Words or sentences with similar meanings end up close together in that space.

### Example
- `"king"` → `[0.23, -1.02, 0.45, ..., 0.11]`  (a vector of, say, 300 numbers)
- `"queen"` → `[0.21, -0.98, 0.47, ..., 0.09]`  (very similar numbers)
- `"apple"` → `[0.65, 0.12, -0.33, ..., 0.72]`  (different from royalty words)

---

## 2. Why Do We Need Embeddings?

Computers don't understand words directly — they need numbers. Traditional methods like **one‑hot encoding** treat every word as independent:

- `"cat"` → `[1,0,0,0,...]`
- `"dog"` → `[0,1,0,0,...]`

In one‑hot, all words are equally different (distance is always the same), so the model can't learn that *cat* and *dog* are both pets.

**Embeddings solve this** by placing words in a continuous space where **distance reflects semantic similarity**.

---

## 3. How Are Embeddings Created?

Embeddings are learned by training neural networks on massive amounts of text. The model learns to predict words from context (or vice versa). Popular algorithms:

- **Word2Vec** (2013) – uses either “skip‑gram” (predict surrounding words from a target) or “CBOW” (predict target from surrounding words).
- **GloVe** (2014) – learns by factorizing word‑co‑occurrence matrices.
- **FastText** – handles out‑of‑vocabulary words by using subword information.
- **Modern transformers** (BERT, GPT) – produce **contextual embeddings**: the same word gets different vectors depending on its context (e.g., “bank” in “river bank” vs. “money bank”).

### Intuition
The model sees millions of sentences like:  
*“I drank a glass of ___.”*  
It learns that the blank is often filled with *water*, *milk*, *juice* — so these words get similar vectors.

---

## 4. Properties of Embeddings

### Dimensionality
Typical sizes: 50, 100, 300, 768, 1024, 1536 (OpenAI's `text-embedding-3-small` uses 1536).  
Higher dimensions can capture more nuance but require more storage.

### Distance Metrics
- **Cosine similarity** – measures the angle between vectors (most common).  
  Range: -1 (opposite) to 1 (identical).  
- **Euclidean distance** – straight‑line distance.  
- **Dot product** – often used when vectors are normalized.

### Vector Arithmetic
Famous example:  
`vector("king") - vector("man") + vector("woman") ≈ vector("queen")`  

This shows that embeddings capture **relationships** (gender, royalty, etc.) in the geometry.

---

## 5. Types of Embeddings

| Type | Description | Example Models |
|------|-------------|----------------|
| **Static** | Each word has one fixed vector, regardless of context. | Word2Vec, GloVe, FastText |
| **Contextual** | The same word gets different vectors based on surrounding text. | BERT, GPT, OpenAI embeddings |

For most modern AI apps (RAG, semantic search), **contextual embeddings** (like from OpenAI or sentence‑transformers) are used because they handle polysemy (multiple meanings).

## 6. Generating Embeddings – Practical Examples

### Using OpenAI API (JavaScript)
```javascript
import OpenAI from 'openai';

const openai = new OpenAI({ apiKey: 'YOUR_API_KEY' });

async function getEmbedding(text) {
  const response = await openai.embeddings.create({
    model: 'text-embedding-3-small',  // or text-embedding-3-large
    input: text,
  });
  return response.data[0].embedding; // array of 1536 numbers
}

// Example
const embedding = await getEmbedding("The quick brown fox jumps over the lazy dog");
console.log(embedding); // [ -0.0123, 0.0456, ..., 0.0789 ]
```

### Using Hugging Face (sentence-transformers) in Python
```python
from sentence_transformers import SentenceTransformer

model = SentenceTransformer('all-MiniLM-L6-v2')  # 384-dimensional
sentences = ["This is an example sentence", "Each sentence is converted"]
embeddings = model.encode(sentences)
print(embeddings.shape)  # (2, 384)
```

## 7. Working with Embeddings – Key Operations

### Compute Similarity
```javascript
function cosineSimilarity(vecA, vecB) {
  const dot = vecA.reduce((sum, a, i) => sum + a * vecB[i], 0);
  const normA = Math.sqrt(vecA.reduce((sum, a) => sum + a * a, 0));
  const normB = Math.sqrt(vecB.reduce((sum, b) => sum + b * b, 0));
  return dot / (normA * normB);
}

const sim = cosineSimilarity(embedding1, embedding2);
console.log(sim); // between -1 and 1
```

### Find Most Similar Items
You can compare a query embedding against many pre‑computed embeddings and return the top‑k closest. This is the basis of **semantic search**.

## 8. Use Cases in AI Applications

- **Semantic Search** – Search by meaning, not keywords.  
  *E.g., query “affordable lodging in Paris” retrieves documents about “cheap hotels near Eiffel Tower” even if they don't contain the exact words.*
- **Retrieval‑Augmented Generation (RAG)** – Convert user question to embedding, find relevant chunks from a vector DB, feed to LLM.
- **Clustering** – Group similar documents (e.g., customer feedback by topic).
- **Classification** – Use embeddings as features for a classifier.
- **Recommendation** – Recommend items with similar embeddings to a user's past interactions.

## 10. Key Takeaways

- **Embeddings = dense vectors** that encode meaning.
- **Similar meaning → similar vectors** (high cosine similarity).
- **Generated by neural networks** trained on large text corpora.
- **Contextual embeddings** are now standard for advanced NLP.
- They are the foundation for **vector databases**, **RAG**, and **semantic search**.
