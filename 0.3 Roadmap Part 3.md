## Phase 2 — Core AI Patterns (Weeks 4-8)

### 2.1 Embeddings & Vector Search
- **2.1.1** What embeddings are
  - Text → dense vector transformation
  - Similarity properties
  - Vector arithmetic (king - man + woman)
- **2.1.2** Generating embeddings
  - OpenAI text-embedding-3-small (1536d)
  - Cohere embed models
  - Batch embedding for efficiency
- **2.1.3** Similarity metrics
  - Cosine similarity (most common)
  - Dot product (normalized vectors)
  - Euclidean distance
- **2.1.4** Chunking strategies
  - Fixed-size with overlap
  - Recursive (hierarchical separators)
  - Semantic (by meaning shift)
  - Token-based (most accurate)
  - Chunk overlap importance (10-20%)
- **2.1.5** Chunk quality
  - Optimal chunk size (256-512 tokens)
  - Context continuity
  - Boundary detection

### 2.2 Vector Databases
- **2.2.1** Pinecone

  - Namespace isolation
  - Metadata filtering
  - Cost model
- **2.2.2** Supabase pgvector
  - PostgreSQL native integration
  - Hybrid search capability
  - Row-level security
  - Best for MERN stacks
- **2.2.3** Weaviate
  - GraphQL interface
  - Built-in vectorization
  - Modular AI integrations
- **2.2.4** Qdrant

- **2.2.5** Chroma

- **2.2.6** Selection criteria
  - Managed vs self-hosted
  - Cost vs control vs scale
  - MERN stack compatibility

### 2.3 RAG (Retrieval-Augmented Generation)
- **2.3.1** Basic RAG pipeline
  - Upload → chunk → embed → store → retrieve → generate
  - Query embedding generation
  - Top-k retrieval
  - Context assembly
- **2.3.2** Advanced RAG techniques
  - **HyDE**: Hypothetical Document Embeddings
  - **Re-ranking**: Cross-encoders, Cohere Rerank
  - **Multi-query retrieval**: Query variants generation
  - **Contextual compression**: Trimming irrelevant parts
  - **Parent-child chunking**: Small chunks for search, full docs for context
  - **Hybrid search**: Vector + BM25 + RRF
  - **Self-querying retrievers**: LLM generates metadata filters
- **2.3.3** RAG failure modes
  - Chunking as #1 failure point
  - Retrieval gaps
  - Context overflow
  - Irrelevant chunk inclusion
